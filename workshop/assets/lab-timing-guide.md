# Lab Timing Guide

## Facilitator assumptions

- **Format:** Two-day instructor-led Copilot Studio workshop for a mixed audience of makers, IT pros, and developers.
- **Suggested day length:** 9:00–16:30 local time.
- **Core workshop rule:** Keep the shared path on generally available capabilities, with **GPT-5 Chat** as the recommended hands-on model. Mention GPT-4.1 only as a comparison point when participants need platform context.
- **Target pacing:** Aim for roughly **40/60 demo-to-hands-on on Day 1** and **35/65 on Day 2**.
- **Setup buffers:** Protect the opening **20 minutes on Day 1** and **15 minutes on Day 2** for sign-in, environment fixes, and room alignment.
- **MCP:** GA capability — use the in-product onboarding wizard only, no preview paths.

## Fast facilitation rules

- **Demo once, then release:** Aim for 5–8 minutes of demo followed by a visible hands-on timebox.
- **Checkpoint before every break:** Put the current lab number and restart point on screen before the room disperses.
- **Protect core outcomes first:** If time slips, preserve Labs 06, 18, 22, and 24 before optional depth.
- **Use role-based pairing:** Pair a maker with an IT pro or developer whenever access, governance, or extensibility questions surface.

## Quick recovery playbook

| Situation | Recommended move |
| --- | --- |
| More than 20% of the room has sign-in or environment issues | Pause independent work, solve access by cohort, and keep one facilitator screen on the target end state. |
| Dataverse, SharePoint, MCP wizard, or evaluation is blocked for many attendees | Convert that module to facilitator demo mode immediately and protect the rest of the day. |
| The class is ahead of schedule | Add extra test prompts, peer demos, or short architecture discussions instead of new scope. |
| Mixed skill levels are widening | Use buddy pairs, publish clear checkpoints, and narrate the intended done state before each release to hands-on. |
| Lab 24 runs short | Use remaining time for a second evaluation iteration with a revised prompt set rather than moving to Lab 25 early. |
| Lab 25 demand is high | Direct developers to start in parallel at 16:00 only; non-developers remain in the main room for ROI analytics and rollout planning. |
