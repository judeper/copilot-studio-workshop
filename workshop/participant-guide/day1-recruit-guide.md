Day 1 is your foundation day. The goal is to help every participant leave with a working understanding of how Copilot Studio agents are structured, built, tested, and published.

Work through the day in this order:

1. Understand what agents are and where Copilot Studio fits.
2. Learn the core building blocks of agent behavior and grounding.
3. Explore declarative patterns and when they fit the scenario.
4. Package work in solutions and review prebuilt agents as reusable accelerators.
5. Create or refine a custom agent with knowledge attached.
6. Add interaction patterns with topics, Adaptive Cards, flows, and triggers.
7. Review publishing, licensing and credits, ROI talking points, and readiness for the next day.

## Expected outputs

By the end of Day 1, you should have:

- A clear understanding of the difference between core agent concepts and implementation choices.
- Experience creating or modifying an agent in Copilot Studio.
- A grounded agent that can respond from provided source content.
- Exposure to solution packaging, prebuilt agent patterns, and the Day 1 licensing and ROI close.
- Exposure to topics, cards, flows, triggers, and publishing workflow.
- A clear sense of whether you are ready for Day 2 advanced modules.

## Lab flow

### Module 1: Orientation and concepts

Focus on the business value of agents, not just the definitions. Pay attention to how grounding, orchestration, and user interaction all fit together.

### Module 2: Foundations in Copilot Studio

Use this section to learn where the main authoring surfaces live and what each one controls. If the interface is new to you, prioritize understanding the flow over memorizing every menu.

### Module 3: Declarative patterns, solutions, and prebuilt agents

Observe how simple prompt-driven experiences differ from more tailored agents. Use the solutions and prebuilt-agent labs to understand what can be packaged, reused, and accelerated before you start from scratch.

### Module 4: Custom agent and grounding

Build toward a working agent experience by creating or refining a custom agent and attaching knowledge that supports grounded responses.

### Module 5: Topics, cards, flows, and triggers

Use this module to connect conversation with action. Focus on what input the agent collects, what process it triggers, and how success is validated.

### Module 6: Publish and reflect

Finish by understanding how a workshop prototype differs from a production-ready asset. Watch for environment, access, and governance considerations called out by your facilitator.

### Module 7: Licensing, credits, ROI, and Day 2 prep

Close by reviewing Copilot Credits, user licensing versus capacity consumption, and the rollout or ROI questions your team would need to answer before scaling the scenario.

> **Tip:** If you fall behind, aim to complete one strong end-to-end path rather than skimming every screen without a working result.

## Audience notes

- [Maker] Focus on clarity of instructions, prompts, and user experience.
- [IT Pro] Focus on environment setup, access, publishing permissions, and governance implications.
- [Developer] Focus on solution structure, extensibility, and patterns you would automate or formalize later.

## End-of-day recap

At the close of Day 1, make sure you can answer these questions:

- What makes an agent grounded and useful?
- Which Copilot Studio features did you use directly today?
- Which parts of the build require environment or governance planning?
- What commercial or readiness questions still need answers before production rollout?
- Are you ready to continue into the enterprise Hiring Agent scenario tomorrow?

If your facilitator is mapping the day to the Recruit path, use the day’s completed modules as your checkpoint toward the Recruit badge or equivalent readiness. If you still feel uncertain about the Day 1 foundations, revisit the linked labs before starting Day 2.
