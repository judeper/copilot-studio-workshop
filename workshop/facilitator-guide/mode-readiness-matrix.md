# Workshop Mode Readiness Matrix

The Copilot Studio workshop runs in four distinct operational modes that you should never mix in the same Power Platform environment. **StudentReady** prepares the shared prerequisites that every participant depends on (SharePoint site, AgentCreators security group, baseline tenant settings). **FacilitatorDemo** stages a facilitator-only environment where the WoodgroveLending solution and Lab 13 base data can be pre-imported so the speaker has a working scenario to demo from. **Per-student provisioning** stamps out one isolated sandbox environment per participant for the hands-on labs. **Fallback rebuild** is the advanced facilitator-only rescue path that copies a completed gold-source environment, repairs bindings, and validates the result so a working demo is available if the primary environment fails.

| Mode | Setup Command | Validation Command | Expected Artifacts | Manual Checks | EnvironmentPurpose |
| --- | --- | --- | --- | --- | --- |
| StudentReady (shared prerequisites) | `Invoke-WorkshopLabSetup.ps1 -Mode StudentReady` | `Invoke-WorkshopPrereqCheck.ps1` | Shared SharePoint site, AgentCreators security group, shared environment baseline | Verify SharePoint lists exist; verify the AgentCreators group has at least one member | `Shared` |
| FacilitatorDemo (solution-only) | `Invoke-WorkshopLabSetup.ps1 -Mode FacilitatorDemo` then `Import-WorkshopEnterpriseAssets.ps1 -ImportSolution` | `Import-WorkshopEnterpriseAssets.ps1` (no flags = validate only) | WoodgroveLending solution imported (no records yet) | Open the **Woodgrove Lending Hub** app — confirm tables exist (no records yet) | `FacilitatorDemo` |
| FacilitatorDemo (demo-ready) | Same as above, plus `Import-WorkshopEnterpriseAssets.ps1 -ImportBaseData` | FetchXML spot checks via `Invoke-WorkshopFacilitatorFallbackValidation.ps1` | Loan types, assessment criteria, sample applicants, and sample loan applications seeded | Open **Woodgrove Lending Hub** — confirm at least 2 sample applications visible | `FacilitatorDemo` |
| Per-student provisioning | `Invoke-StudentEnvironmentProvisioning.ps1` | `Invoke-StudentEnvironmentProvisioning.ps1 -ValidateOnly` | N student sandbox environments + per-student SharePoint sites + Teams teams + Copilot credits | Spot-check 1 student environment via Power Platform Admin Center; confirm credits ≥ 25K | `Student` |
| Fallback rebuild | `Set-WorkshopFacilitatorFallbackSource.ps1 -UpdateConfig` then `Export-WorkshopFacilitatorArtifactLayers.ps1` then `Invoke-WorkshopFacilitatorFallbackBuild.ps1` then `Invoke-WorkshopFacilitatorBindingRepair.ps1` | `Invoke-WorkshopFacilitatorFallbackValidation.ps1` | Cloned facilitator-only environment with connection references and environment variables repaired | Test the agent in the cloned environment responds to a sample query | `Fallback` |

## EnvironmentPurpose guard

Each mode pins its environment to a specific `Workshop.EnvironmentPurpose` value in `workshop-config.json`. Valid values are `Student`, `FacilitatorDemo`, `Fallback`, and `Shared`. The shared `Assert-FacilitatorOnlyEnvironment` helper in `Common.ps1` is invoked by every facilitator-only script (`Import-WorkshopEnterpriseAssets.ps1`, `Set-WorkshopFacilitatorFallbackSource.ps1`, `Export-WorkshopFacilitatorArtifactLayers.ps1`, `Invoke-WorkshopFacilitatorBindingRepair.ps1`, `Invoke-WorkshopFacilitatorFallbackBuild.ps1`, `Invoke-WorkshopFacilitatorFallbackValidation.ps1`, `Remove-WorkshopFacilitatorEnvironment.ps1`) immediately after the config is loaded. Those scripts refuse to run unless `EnvironmentPurpose` is `FacilitatorDemo` or `Fallback`, which prevents an accidental run against a student-shared environment from importing demo data, exporting artifacts, or deleting resources that students depend on. If you legitimately need to repurpose an environment, update `Workshop.EnvironmentPurpose` in the config file before re-running the script.
